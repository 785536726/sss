HTTP响应一般由三部分构成
//		1.数字和文字组成的状态码,用来显示成功或者失败
//		2.响应头,响应头和请求头一样包含许多有用的信息,例如服务器类型,日期时间,内容类型
//		3.响应体,也就是正文

HTTP请求一般由四部分构成
//		1.HTTP请求的方法或者动作,比如get或者post
//		2.正在请求的url,也就是请求的路径
//		3.请求头,包含客户端环境信息,身份验证信息等
//		4.请求正文,包含客户提交的查询字符串信息,表单信息等

get一般用于信息获取,使用url传递参数,对发送的信息数量也有限制,一般在2000字符
//		post一般用于修改服务器上的资源,对发送信息的数量没有限制

状态码
//		100-199 用于指定客户端应相应的某些动作。 
//		200-299 用于表示请求成功。 
//		300-399 用于已经移动的文件并且常被包含在定位头信息中指定新的地址信息。 
//		400-499 用于指出客户端的错误。 
//		500-599 用于支持服务器错误。

//		readyState	
//		0:请求未初始化,open还没有调用
//		1:服务器连接已经建立,open已经调用了
//		2:请求已经接收了,也就是接收到头信息了
//		3:请求处理中,也就是接收到响应主体了
//		4:请求已经完成,且响应已经就绪,也就是响应完成了
